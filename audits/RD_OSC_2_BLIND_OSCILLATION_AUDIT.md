# RD-OSC.2 — Blind Oscillation Audit

**Goal:** Determine whether the oscillation between explanatory types emerges from blind coding, not from supplied categories.

**Method:** Take 20 studies. Blind the study names. Blind previous classifications. Ask multiple independent coders: "What changed between the beginning and end of this study?" Only after coding, cluster the changes. Then ask: Does an oscillatory structure emerge?

**Critical Correction:** Do not ask about "explanatory focus" — that smuggles in explanation. Ask only what changed. Then later ask: "Did explanation increase?" This separation prevents explanation from being injected into the coding process.

---

## Procedure

### Phase 1: Preparation

1. Select 20 studies from the RD archive
2. Blind study names (replace with random identifiers)
3. Blind previous classifications (remove all type labels)
4. Prepare coding packets for independent coders

### Phase 2: Blind Coding

1. Recruit multiple independent coders
2. Each coder receives identical blind packets
3. Each coder answers: "What changed between the beginning and end of this study?"
4. Coders use their own vocabulary (no predefined categories)

### Phase 3: Analysis

1. Collect all coder responses
2. Cluster the changes using open coding
3. Determine if oscillatory structure emerges
4. Compare across coders
5. **Separately** ask: "Did explanation increase?"

---

## Questions

1. Does an oscillatory structure emerge from blind coding?
2. If yes, what are the types?
3. If no, is the oscillation a cognitive artifact?
4. Did explanation increase?
